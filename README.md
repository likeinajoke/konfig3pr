# Практика 4. Ассемблер УВМ

## Этап 1

**Цель этапа:**  
Перевести текстовую программу на простом ассемблерном языке в *промежуточное представление (IR)*.

IR — это список инструкций, каждая представлена словарём:

- `mnemonic` — имя команды
- `A` — код инструкции
- `B`, `C`, `D`, `CONST` — аргументы инструкции (если есть)

---

## Синтаксис языка ассемблера

### Общие правила
- Одна инструкция — одна строка.
- Операнды разделяются запятыми.
- Комментарии обозначаются `;` или `#`.
- Поддерживаются числа:
  - десятичные
  - шестнадцатеричные (`0x...`)

### Поддерживаемые инструкции

#### `LOAD_CONST <CONST>, <ADDRESS>`
Загрузить значение в память.

#### `READ <B_addr>, <C_addr>`
Чтение из ячейки памяти в другую.

#### `WRITE <B_addr>, <C_addr>`
Запись значения из одного адреса в другой.

#### `MIN <B_addr>, <C_addr>, <D_addr>`
Бинарная операция: вычисление минимума двух значений.
